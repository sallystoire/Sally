# Paname City

A Clash of Clans-style Discord Activity game built for the discord.gg/paname server (Guild ID: 1489787998676713632).

## Architecture

- **Frontend**: React + Vite at `artifacts/paname-city/` (served at `/`)
- **Backend**: Express API at `artifacts/api-server/` (served at `/api`)
- **Database**: PostgreSQL via Drizzle ORM at `lib/db/`
- **API Contract**: OpenAPI spec + Orval codegen at `lib/api-spec/`
- **Discord Bot**: discord.js bot at `artifacts/api-server/src/bot/`
- **Auth**: Discord OAuth2 + JWT stored in localStorage

## Game Features

- Kingdom building with 9 building types (townhall, goldmine, farm, lumbermill, quarry, barracks, cannon, wall)
- Resource collection (gold, wood, stone, food) with passive generation over time
- Building upgrades with resource costs
- Raiding other players' kingdoms (attack vs defense power calculation)
- Shop with resource packs, shields, and boost items
- Promo codes (created via Discord bot `/code` command)
- Leaderboard ranked by trophies

## Discord Bot Commands

All commands are ephemeral (only visible to the caller):
- `/code <ressource> <montant>` — Create a promo code (sys only)
- `/sys <membre>` — Add a sys member (sys only)
- `/unsys <membre>` — Remove a sys member (sys only)
- `/syslist` — Show all sys members

## Deployment (Railway)

The app is fully configured for Railway deployment via `railway.toml` and `Dockerfile`.

### Railway Variables Required (set manually in Railway Dashboard)
- `DISCORD_CLIENT_ID` — Discord application client ID
- `DISCORD_CLIENT_SECRET` — Discord application client secret
- `DISCORD_BOT_TOKEN` — Discord bot token
- `DISCORD_GUILD_ID=1489787998676713632` — Paname server guild ID
- `DATABASE_URL` — Auto-set by Railway Postgres plugin (`${{Postgres.DATABASE_URL}}`)
- `SESSION_SECRET` — Auto-generated by railway.toml (`${{secret(32)}}`)

### Build Notes
- The Dockerfile builds the frontend first (needs `DISCORD_CLIENT_ID` as build ARG for `VITE_DISCORD_CLIENT_ID`), then the backend
- Single service serves both frontend (static) and API

## Database Schema

Tables: `users`, `kingdoms`, `buildings`, `raids`, `shop_items`, `promo_codes`, `code_redemptions`, `sys_users`

## Development

```bash
# Push DB schema
pnpm --filter @workspace/db run push

# Run codegen after changing OpenAPI spec
pnpm --filter @workspace/api-spec run codegen

# Start API server
pnpm --filter @workspace/api-server run dev

# Start frontend
pnpm --filter @workspace/paname-city run dev
```
